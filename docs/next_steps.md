## 三、用户体验问题专项解决方案

### 3.1 问题描述

用户反馈场景：
- **跳转广告**：点进签到页面后出现广告跳转
- **页面加载慢**：签到页面加载时间过长
- **登录态丢失**：签到过程中登录状态丢失
- **UI 变化**：平台 UI 改版导致签到失败

### 3.2 问题识别机制

```python
# autosignin/core/page_analyzer.py

class PageAnalyzer:
    """页面内容分析器"""

    def __init__(self):
        self.ad_patterns = [
            "跳转中",
            "正在跳转",
            "loading",
            "ad.",
            "advertisement",
            "sponsor",
            "推广",
            "广告",
        ]

        self.error_patterns = [
            "页面不存在",
            "404",
            "403",
            "访问被拒绝",
            "请登录",
            "登录失效",
        ]

    async def analyze(self, response: httpx.Response, platform: str) -> AnalysisResult:
        """分析页面内容"""
        result = AnalysisResult(platform=platform)

        content = response.text.lower()
        url = str(response.url).lower()

        # 检测广告跳转
        if self._detect_ad_redirect(response):
            result.has_ad_redirect = True
            result.issues.append("检测到广告跳转")

        # 检测登录失效
        if self._detect_login_expired(content, url):
            result.login_expired = True
            result.issues.append("登录状态已失效")

        # 检测页面加载失败
        if self._detect_loading_failure(content):
            result.loading_failed = True
            result.issues.append("页面加载失败")

        # 检测 UI 变化
        if await self._detect_ui_change(response):
            result.ui_changed = True
            result.issues.append("页面 UI 可能已变更")

        return result

    def _detect_ad_redirect(self, response: httpx.Response) -> bool:
        """检测广告跳转"""
        # 检查 URL 是否包含第三方跳转参数
        redirect_params = ["url=", "redirect=", "target="]
        for param in redirect_params:
            if param in str(response.url):
                return True

        # 检查响应时间是否异常（广告页面通常很快）
        if response.elapsed.total_seconds() < 0.5:
            content = response.text.lower()
            for pattern in self.ad_patterns:
                if pattern.lower() in content:
                    return True

        return False

    def _detect_login_expired(self, content: str, url: str) -> bool:
        """检测登录失效"""
        login_indicators = [
            "登录",
            "login",
            "请先登录",
            "请登录",
            "未登录",
            "授权",
            "authorize",
        ]

        current_url = url.lower()
        for indicator in login_indicators:
            if indicator in current_url:
                return True

        # 检查是否跳转到登录页
        login_urls = ["passport", "login", "signin", "auth"]
        for login_url in login_urls:
            if login_url in current_url:
                return True

        return False

    async def _detect_ui_change(self, response: httpx.Response) -> bool:
        """检测 UI 变化"""
        # 通过对比关键元素是否存在来判断 UI 是否变化
        critical_elements = await self._find_critical_elements(response)
        return len(critical_elements) < 3  # 关键元素少于 3 个认为 UI 变化
```

### 3.3 解决方案矩阵

| 问题类型 | 识别方法 | 自动处理 | 人工处理 | 预防措施 |
|---------|---------|---------|---------|---------|
| **跳转广告** | URL 参数分析 + 内容检测 | 记录并跳过 | 检查目标平台状态 | 降低请求频率 |
| **登录失效** | 登录状态 API 检测 | 自动告警 | 更新 Cookie | Cookie 过期前提醒 |
| **页面加载慢** | 响应时间监控 | 增加超时时间 | 检查网络 | 使用 CDN 节点 |
| **UI 变更** | 关键元素检测 | 暂停平台 + 告警 | 手动确认修复 | 版本锁定 + 监控 |
| **验证码拦截** | 状态码 + 内容检测 | 暂停任务 + 告警 | 人工处理 | 降低频率 + 代理 |
| **IP 被封** | 连续 403/429 检测 | 切换代理 | 联系平台 | 使用代理池 |

## 五、验收标准细化

### 5.1 功能验收

| 验收点 | 验收条件 | 测试方法 | 通过标准 |
|-------|---------|---------|---------|
| 单次签到 | 能成功签到 | `python main.py sign` | 返回 success=true |
| 指定平台 | 能指定平台签到 | `python main.py sign -p bilibili` | 只执行 bilibili |
| 多平台 | 能同时签到多个 | `python main.py sign -p bilibili zhihu` | 2 个平台都执行 |
| 定时任务 | 能定时执行 | 启动 scheduler 等待 | 按时执行，日志正确 |
| 通知发送 | 能收到通知 | 触发签到 | 通知渠道收到消息 |
| 重复检查 | 能跳过已签到 | 连续执行两次 | 第二次提示已签到 |
| 强制签到 | 能强制执行 | `python main.py sign -f` | 即使已签到也执行 |

### 5.2 异常处理验收

| 异常场景 | 预期行为 | 测试方法 | 通过标准 |
|---------|---------|---------|---------|
| Cookie 无效 | 返回 AuthError | 使用过期 Cookie | 错误消息明确 |
| 网络超时 | 自动重试 | 断网测试 | 重试 3 次后失败 |
| 验证码拦截 | 暂停并告警 | 触发验证码 | 任务暂停，收到通知 |
| 平台 API 错误 | 返回具体错误 | Mock 平台错误 | 错误类型明确 |
| 限流触发 | 退避等待 | 频繁请求 | 等待后自动恢复 |

### 5.3 性能验收

| 指标 | 目标值 | 测量方法 | 通过标准 |
|------|--------|---------|---------|
| 启动时间 | < 3s | `time python main.py status` | 输出时间 < 3s |
| 签到耗时 | < 30s/平台 | 记录总耗时 | 每平台平均 < 30s |
| 内存占用 | < 200MB | 监控内存 | 签到过程中 < 200MB |
| CPU 占用 | < 50% | 监控 CPU | 签到过程中 < 50% |

### 6.3 告警等级定义

| 等级 | 名称 | 触发条件 | 通知方式 | 处理时限 |
|------|------|---------|---------|---------|
| **P0** | 紧急 | 系统无法启动、全部平台签到失败 | 立即通知 | 30 分钟内 |
| **P1** | 严重 | 单平台持续失败、认证批量失效 | 立即通知 | 2 小时内 |
| **P2** | 一般 | 非核心功能异常、偶发失败 | 工作时间通知 | 24 小时内 |
| **P3** | 提示 | 警告信息、性能波动 | 次日汇总 | 72 小时内 |

## 七、风险应对预案

### 7.1 高概率风险应对

| 风险 | 概率 | 影响 | 应对措施 | 恢复时间 |
|------|------|------|---------|---------|
| Cookie 过期 | 高 | 中 | 1. 提前 24h 提醒<br>2. 自动禁用过期账号<br>3. 记录失效时间 | 即时 |
| 验证码拦截 | 高 | 中 | 1. 检测到暂停任务<br>2. 发送告警<br>3. 降低请求频率 | 人工处理 |
| 平台 API 变更 | 中 | 高 | 1. 版本监控告警<br>2. 快速迭代修复<br>3. 降级提示用户 | 24h 内 |

### 7.2 高影响风险应对

| 风险 | 概率 | 影响 | 应对措施 | 恢复时间 |
|------|------|------|---------|---------|
| 账号被封 | 低 | 高 | 1. 自动禁用账号<br>2. 通知用户<br>3. 记录封禁原因 | 人工处理 |
| 数据丢失 | 低 | 高 | 1. 定期备份<br>2. WAL 模式<br>3. 操作日志 | 1h 内 |
| 服务中断 | 低 | 高 | 1. Docker 重启<br>2. 健康检查自愈<br>3. 告警升级 | 30 分钟内 |

### 7.3 降级策略

| 场景 | 降级措施 | 降级后状态 |
|------|---------|-----------|
| 通知渠道全部不可用 | 写入本地日志 | 可查询本地日志 |
| 单平台持续失败 | 熔断该平台 | 其他平台继续 |
| 验证码无法处理 | 暂停任务 + 告警 | 等待人工处理 |
| 数据库不可用 | 内存缓存 | 恢复后同步 |
